## What You Can Contribute

We welcome:
- Bug fixes
- Performance improvements
- Refactoring and cleanup
- Documentation improvements
- Tests
- Accessibility improvements

Please keep contributions focused and reasonably scoped.

---

## What We Do Not Accept

The following will not be accepted:
- Contributions copied from third-party sources with incompatible licenses
- Code that introduces copyleft or viral licenses (e.g., GPL, AGPL)
- Contributions that attempt to change or weaken the project’s license
- Contributions intended primarily to enable unauthorized redistribution

---

## Development Guidelines

- Follow existing code style and architecture
- Write clear commit messages
- Prefer small, reviewable pull requests
- Include tests where applicable
- Avoid introducing new dependencies without discussion

---

## Submitting a Pull Request

1. Fork the repository
2. Create a feature or fix branch
3. Make your changes
4. Ensure the project builds and tests pass
5. Open a pull request with a clear description of the change

By opening a pull request, you confirm that you have read and agree to this
CONTRIBUTING document.

---

## Questions

If you have questions about licensing or acceptable contributions, please open
an issue before starting work.
